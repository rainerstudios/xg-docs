<Info>
  The Subdomain Manager lets you create easy-to-remember addresses for your server, making it simpler for players to connect.
</Info>

## Creating a Subdomain

<Steps>
  <Step title="Access Subdomain Manager">
    Navigate to the **Subdomain Manager** page in your panel
  </Step>
  
  <Step title="Choose Your Prefix">
    Enter your desired subdomain prefix in the text box
    
    <CodeGroup>
      ```text Good Examples
      play
      mc
      server
      myguild
      survival
      ```
      
      ```text Avoid These
      www (reserved)
      admin (reserved)
      api (reserved)
      mail (reserved)
      ```
    </CodeGroup>
  </Step>
  
  <Step title="Select Domain">
    Choose from available domain options in the dropdown menu
  </Step>
  
  <Step title="Create Subdomain">
    Click the **Create** button to generate your custom address
  </Step>
</Steps>

## Using Your Subdomain

<Tabs>
  <Tab title="Minecraft Servers">
    <Steps>
      <Step title="Share with Players">
        Your players can now connect using your custom address:
        
        ```text Example
        play.mcserver.net
        ```
      </Step>
      
      <Step title="Update Server MOTD">
        Update your server's message of the day:
        
        ```properties server.properties
        motd=Welcome to play.mcserver.net!
        ```
      </Step>
      
      <Step title="Configure Server Lists">
        Use your subdomain when submitting to server lists for consistent branding
      </Step>
    </Steps>
  </Tab>
  
  <Tab title="Other Game Servers">
    <Steps>
      <Step title="Game-Specific Setup">
        Configure based on your game type:
        
        <CodeGroup>
          ```cfg CS2/Garry's Mod
          hostname "My Server - play.example.net"
          ```
          
          ```cfg Rust
          server.hostname "My Rust Server [play.example.net]"
          ```
          
          ```cfg Valheim
          -name "My World - play.example.net"
          ```
        </CodeGroup>
      </Step>
      
      <Step title="Update Launch Parameters">
        Include your subdomain in server descriptions and connection info
      </Step>
    </Steps>
  </Tab>
  
  <Tab title="FiveM Servers">
    <Steps>
      <Step title="Update server.cfg">
        ```cfg server.cfg
        sv_projectName "My Server"
        sv_projectDesc "Connect at play.fivem.net"
        ```
      </Step>
      
      <Step title="FiveM Server List">
        Update your server listing with the new subdomain for professional appearance
      </Step>
    </Steps>
  </Tab>
</Tabs>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Subdomain Not Working" icon="exclamation-triangle">
    **Problem**: Players can't connect using the subdomain
    
    **Solutions**:
    1. Wait 5-10 minutes for DNS propagation
    2. Clear DNS cache: `ipconfig /flushdns` (Windows) or `sudo dscacheutil -flushcache` (Mac)
    3. Try connecting with the original IP to confirm server is online
    4. Check if subdomain was created successfully in the panel
  </Accordion>
  
  <Accordion title="Subdomain Already Taken" icon="xmark-circle">
    **Problem**: Your desired subdomain is unavailable
    
    **Solutions**:
    - Try variations: `play1`, `play-server`, `myplay`
    - Use your server name: `myserver`, `myguild`
    - Try different domain options
    - Add descriptive words: `survival-server`, `creative-world`
  </Accordion>
  
  <Accordion title="DNS Propagation Delays" icon="clock">
    **Problem**: Subdomain works sometimes but not always
    
    **Solutions**:
    - DNS changes can take up to 24 hours globally
    - Test from different networks/devices
    - Use online DNS checkers to verify propagation
    - Be patient - propagation varies by location
  </Accordion>
</AccordionGroup>

## Monitoring Your Subdomain

<Steps>
  <Step title="Test Connectivity">
    Regularly test your subdomain from different locations and networks
  </Step>
  
  <Step title="Monitor DNS Health">
    Use tools like [DNSChecker.org](https://dnschecker.org) to verify global propagation
  </Step>
  
  <Step title="Update Marketing Materials">
    Ensure all promotional materials use the new subdomain consistently
  </Step>
</Steps>

<Warning>
  Never delete a subdomain that players are actively using without giving advance notice and providing the new connection details.
</Warning>
