This comprehensive guide walks you through setting up your Minecraft server on XGamingServer from initial configuration to welcoming your first players.

<Info>

## Step 2: Configure Resources

Proper resource allocation ensures smooth gameplay for your players:

### Resource Allocation Guidelines

<Tabs>

  <Tab title="Modded Servers">
    **Recommended for modded gameplay:**
    
    | Mod Count | CPU Cores | RAM | Disk Space |
    |-----------|-----------|-----|------------|

    | 50-150    | 3 cores   | 8GB | 20GB       |

<Warning>

</Warning>

## Step 3: Basic Server Configuration

### Essential server.properties Settings

Navigate to **File Manager** and edit the `server.properties` file:

```properties
# Server Identity
server-name=My Awesome Server
motd=Welcome to my Minecraft server!

# Basic Settings  
gamemode=survival
difficulty=normal
max-players=20

# World Generation
level-name=world
level-type=minecraft:normal
generate-structures=true

# Performance Settings
view-distance=10
simulation-distance=8

# Security Settings
online-mode=true
enable-whitelist=false

# Networking
server-port=25565
query.port=25565
```

<AccordionGroup>
  <Accordion title="Gamemode Options">
    - **survival**: Players must gather resources and can die
    - **creative**: Players have unlimited resources and can fly  
    - **adventure**: Custom maps with limited block interaction
    - **spectator**: Players can observe but not interact
  </Accordion>
  
  <Accordion title="Difficulty Levels">
    - **peaceful**: No hostile mobs, health regenerates
    - **easy**: Reduced damage from mobs and environment
    - **normal**: Standard gameplay balance
    - **hard**: Maximum challenge with full damage
  </Accordion>
  
  <Accordion title="Performance Considerations">
    - **view-distance**: Higher values = more world loaded = more RAM/CPU usage
    - **simulation-distance**: Affects mob spawning and crop growth radius
    - **max-players**: More players = higher resource requirements
  </Accordion>
</AccordionGroup>

## Step 4: Security & User Management

### Setting Up Operators

Grant yourself admin permissions:

<Steps>
  <Step title="Start Your Server">
    Use the **Console** tab to boot up your server
  </Step>
  
  <Step title="Add Yourself as Operator">
    In the console, type:
    ```
    op YourMinecraftUsername
    ```
  </Step>
  
  <Step title="Verify Permissions">
    Join the server and test admin commands like `/gamemode creative`
  </Step>
</Steps>

### Whitelist Configuration (Recommended)

For better control over who can join:

```bash
# Enable whitelist
whitelist on

# Add players
whitelist add PlayerName
whitelist add AnotherPlayer

# View whitelist
whitelist list
```

## Step 6: World Management

### Creating Custom Worlds

<Tabs>
  <Tab title="Upload Existing World">
    <Steps>
      <Step title="Prepare World Folder">
        Ensure your world folder contains `level.dat` and region files
      </Step>
      
      <Step title="Stop Server">
        Use the console to stop your server safely
      </Step>
      
      <Step title="Upload via File Manager">
        Navigate to your server root and upload the world folder
      </Step>
      
      <Step title="Update server.properties">
        Change `level-name=YourWorldName` to match the uploaded folder
      </Step>
      
      <Step title="Start Server">
        Boot up the server to load your custom world
      </Step>
    </Steps>
  </Tab>
  
  <Tab title="Generate New World">
    <Steps>
      <Step title="Stop Server">
        Always stop the server before changing world settings
      </Step>
      
      <Step title="Edit World Settings">
        In `server.properties`, modify:
        ```properties
        level-name=NewWorldName
        level-type=minecraft:amplified  # or flat, normal, etc.
        level-seed=YourSeedHere
        ```
      </Step>
      
      <Step title="Delete Old World (Optional)">
        Remove the old world folder to free up space
      </Step>
      
      <Step title="Start Server">
        The server will generate a new world based on your settings
      </Step>
    </Steps>
  </Tab>
</Tabs>

## Step 7: Performance Optimization

### Java Flags (Aikar's Flags)

For optimal performance, configure Java startup flags:

<CodeGroup>
  ```bash 2-4GB RAM
  java -Xms2G -Xmx2G -XX:+UseG1GC -XX:+ParallelRefProcEnabled -XX:MaxGCPauseMillis=200 -XX:+UnlockExperimentalVMOptions -XX:+DisableExplicitGC -XX:+AlwaysPreTouch -XX:G1NewSizePercent=30 -XX:G1MaxNewSizePercent=40 -XX:G1HeapRegionSize=8M -XX:G1ReservePercent=20 -XX:G1HeapWastePercent=5 -XX:G1MixedGCCountTarget=4 -XX:InitiatingHeapOccupancyPercent=15 -XX:G1MixedGCLiveThresholdPercent=90 -XX:G1RSetUpdatingPauseTimePercent=5 -XX:SurvivorRatio=32 -XX:+PerfDisableSharedMem -XX:MaxTenuringThreshold=1 -Dusing.aikars.flags=https://mcflags.emc.gs -Daikars.new.flags=true -jar server.jar nogui
  ```

## Step 8: Backup Strategy

### Automated Backup Setup

<Steps>
  <Step title="Navigate to Schedules">
    Go to the **Schedules** tab in your control panel
  </Step>
  
  <Step title="Create Backup Schedule">
    Set up automated backups:
    - **Frequency**: Daily at 3 AM (off-peak hours)
    - **Action**: Create Backup
    - **Retention**: Keep last 7 backups
  </Step>
  
  <Step title="Test Backup">
    Create a manual backup to ensure the process works
  </Step>
</Steps>

<Tip>
Store critical backups offline (download them locally) for extra protection against data loss.
</Tip>

## Troubleshooting Common Setup Issues

<AccordionGroup>
  <Accordion title="Server Won't Start" icon="circle-exclamation">
    **Common causes:**
    - Insufficient RAM allocation
    - Corrupted world files
    - Plugin conflicts
    - Invalid server.properties settings
    
    **Solutions:**
    1. Check console logs for specific error messages
    2. Increase RAM allocation in server settings
    3. Remove recently added plugins
    4. Reset server.properties to default values
  </Accordion>
  
  <Accordion title="Poor Performance" icon="gauge-low">
    **Symptoms:**
    - TPS below 20
    - Player lag and connection timeouts
    - High memory usage
    
    **Solutions:**
    1. Apply Aikar's Java flags
    2. Reduce view-distance and simulation-distance
    3. Limit maximum players
    4. Remove resource-intensive plugins
    5. Optimize world settings
  </Accordion>
  
  <Accordion title="Players Cannot Connect" icon="plug-circle-exclamation">
    **Troubleshooting steps:**
    1. Verify server is running and shows "Done!" in console
    2. Check if server IP and port are correct
    3. Ensure online-mode matches player account type
    4. Test whitelist settings if enabled
    5. Check for firewall or network issues
  </Accordion>
</AccordionGroup>
